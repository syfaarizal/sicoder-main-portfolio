‚è≥ Synchronous

Runs line by line. No patience.

‚ö° Asynchronous

üìû Callback

A callback is a function called later.

JavaScript

function loadData(callback) {
  setTimeout(() => {
    callback("Data loaded");
  }, 2000);
}

loadData((result) => {
  console.log(result);
});

Works ‚Äî but too many callbacks lead to callback hell.

ü§ù Promise

A promise represents a value that will exist later.

  • pending
  • fulfilled
  • rejected
JavaScript

const getData = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Data loaded");
  } else {
    reject("Failed to load data");
  }
});

getData
  .then(result => console.log(result))
  .catch(error => console.log(error));

Cleaner than callbacks. Still readable.

üöÄ async / await

Looks synchronous. Works asynchronously.

JavaScript

async function fetchData() {
  try {
    const result = await getData;
    console.log(result);
  } catch (error) {
    console.log(error);
  }
}

This is the modern standard.

üåê Fetch API

Fetch is how JavaScript talks to the internet.

JavaScript

fetch("https://api.example.com/data")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.log(err));

Cleaner with async/await:

JavaScript

async function getData() {
  const res = await fetch("https://api.example.com/data");
  const data = await res.json();
  console.log(data);
}
